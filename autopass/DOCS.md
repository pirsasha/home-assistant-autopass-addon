# Автопропуск

Управление постоянными, временными и одноразовыми автомобильными пропусками
внутри Home Assistant.

## Возможности

- распознавание номеров через Frigate LPR;
- управление воротами через сущности Home Assistant;
- владельцы, автомобили и расписания доступа;
- журнал проездов со снимками;
- импорт и экспорт CSV/XLSX;
- роли пользователей и аудит действий;
- резервные копии SQLite;
- светлая и тёмная темы;
- лицензионный каркас дополнительного модуля коммерческой парковки;
- настройка тарифов парковки с бесплатным периодом и суточным максимумом;
- активные парковочные сессии, ручной въезд и печать талона;
- отключаемый автоматический платный въезд неизвестных автомобилей;
- оплата, доплата и автоматическое закрытие сессии на выезде;
- подписанная офлайн-лицензия.

## Автоматический платный въезд

В **Настройки → Коммерческая парковка** можно разрешить автоматическое создание
платной сессии и открытие въезда для неизвестного номера. По умолчанию эта
функция выключена: оператор создаёт сессию вручную и самостоятельно открывает
ворота. Действующие постоянные и временные пропуска работают автоматически
независимо от этой настройки.

## Оплата и выезд

Кассир принимает оплату из карточки активной парковочной сессии. После оплаты
система показывает крайний срок выезда. Настроенная выездная камера проверяет
совпадение номера и срок оплаты, открывает выездные ворота и закрывает сессию
только после успешной команды ворот.

Если срок истёк, система рассчитывает доплату и выдаёт новый срок. Если камера
не распознала грязный номер, оператор может закрыть разрешённую сессию вручную.
Кассир или администратор может выполнить принудительное закрытие с обязательной
причиной. Все операции сохраняются в аудите.

## Кассовые смены и отчёты

Перед первым платежом кассир открывает смену и указывает начальный остаток
наличных. Все новые платежи получают номер этой смены. Текущая X-сводка
показывает количество платежей, общую выручку, наличные, карты и переводы.

При закрытии кассир вводит фактическую сумму наличных. Система фиксирует
ожидаемую сумму, фактическую сумму и расхождение. Закрытая Z-сводка больше не
пересчитывается при последующих операциях.

В разделе отчётности выбирается период. Доступны выручка по способам оплаты,
посещаемость, уникальные автомобили, средняя длительность, текущая загрузка и
журнал завершённых парковок. Реестр закрытых сессий выгружается в CSV или XLSX.
Доступ к отчётам требует разрешения `parking_reports`.

## Операционная устойчивость

В настройках коммерческой парковки задаются вместимость, порог предупреждения
и поведение при заполнении. Режим блокировки запрещает только новый платный
въезд. Автомобили с действующим пропуском продолжают въезжать. Оператор может
создать сессию сверх лимита, явно подтвердив исключение.

Сессии старше заданного количества часов помечаются как требующие проверки.
Аварийное открытие въезда или выезда выполняется только после ввода причины и
сохраняется в аудите.

Физическое поведение ворот при полном отказе Home Assistant обеспечивает
контроллер ворот, реле и резервное питание. Add-on хранит выбранную политику и
инструкцию оператору как эксплуатационный регламент.

Для фотографий парковки настраивается отдельный срок хранения. Старые закрытые
сессии можно автоматически обезличивать без удаления финансовых сумм и
агрегированной статистики.

## Лицензия

После установки откройте **Настройки → Лицензия**, скопируйте ID установки и
передайте его поставщику. Полученный файл лицензии загрузите в том же разделе.

До активации действует демонстрационный период 14 дней и льготный период 7 дней.
После завершения срока журнал, экспорт, резервные копии и ручное открытие
остаются доступны.

## Frigate

Для официального Frigate add-on внутри Home Assistant обычно используется
внутренний адрес `http://ccab4aaf-frigate:5000`.

Во вкладке **Настройки** доступны отдельные точки **Въезд** и **Выезд**. Для
каждой задаются собственные камера Frigate, направление, минимальная точность,
сущность ворот, действие и пауза между открытиями. Старые настройки одной
камеры автоматически переносятся во въезд; выезд включается после заполнения.

Одна MQTT-автоматизация передаёт события всех камер. Add-on сам выбирает нужные
ворота по имени камеры, поэтому отдельная автоматизация для выезда не требуется.

## Подключение к Home Assistant

### 1. API-ключ add-on

Откройте **Настройки → Дополнения → Автопропуск → Конфигурация**, задайте
уникальный `api_key` длиной не менее 32 символов и перезапустите add-on.

Это параметр конфигурации add-on. Добавлять `api_key` в
`configuration.yaml` Home Assistant не нужно.

Добавьте тот же ключ в `/config/secrets.yaml`:

```yaml
autopass_api_key: "уникальный-длинный-ключ"
```

### 2. Команды Home Assistant

Добавьте в `/config/configuration.yaml`:

```yaml
rest_command:
  autopass_check_plate:
    url: "http://86015966-autopass:8099/api/check"
    method: POST
    content_type: "application/json"
    headers:
      X-API-Key: !secret autopass_api_key
    payload: >-
      {
        "plate": {{ plate | tojson }},
        "direction": {{ direction | default("entry") | tojson }},
        "camera": {{ camera | default("vorota") | tojson }},
        "gate": {{ gate | default("Главные ворота") | tojson }},
        "snapshot_url": {{ snapshot_url | default("") | tojson }}
      }

  autopass_frigate_event:
    url: "http://86015966-autopass:8099/api/frigate-event"
    method: POST
    content_type: "application/json"
    headers:
      X-API-Key: !secret autopass_api_key
    payload: >-
      {
        "type": {{ event_type | default("lpr") | tojson }},
        "camera": {{ camera | default("") | tojson }},
        "plate": {{ plate | default("") | tojson }},
        "score": {{ score | default(0) | float }},
        "id": {{ event_id | default("") | tojson }}
      }
```

Если секция `rest_command:` уже существует, добавьте обе команды внутрь неё и
не создавайте второй ключ `rest_command:`.

`86015966-autopass` — постоянное внутреннее имя для установки из репозитория
`https://github.com/pirsasha/home-assistant-autopass-addon`. Его не нужно
вычислять: скопируйте пример без изменений. Проверить имя можно во вкладке
информации add-on.

`local-autopass` используется только для локальной установки исходников в
`/addons/autopass`. Если внутренний DNS недоступен, назначьте порт `8099` во
вкладке **Сеть** add-on и используйте IP Home Assistant, например
`http://192.168.1.10:8099/api/frigate-event`. Не публикуйте этот порт в интернет.

### 3. Автоматизация Frigate

Добавьте в `/config/automations.yaml`:

```yaml
- id: autopass_frigate_gate
  alias: "Автопропуск: Frigate → add-on"
  triggers:
    - trigger: mqtt
      topic: frigate/tracked_object_update
  conditions:
    - condition: template
      value_template: >-
        {% set d = trigger.payload | from_json(default={}) %}
        {{ d.get('type') == 'lpr' }}
  actions:
    - action: rest_command.autopass_frigate_event
      data:
        event_type: "{{ trigger.payload_json.get('type', '') }}"
        camera: "{{ trigger.payload_json.get('camera', '') }}"
        plate: "{{ trigger.payload_json.get('plate', '') }}"
        score: "{{ trigger.payload_json.get('score', 0) }}"
        event_id: "{{ trigger.payload_json.get('id', '') }}"
      response_variable: result
  mode: queued
  max: 10
```

Если в `configuration.yaml` уже есть стандартная строка
`automation: !include automations.yaml`, повторно добавлять её не нужно.

После изменений выполните проверку конфигурации в
**Инструменты разработчика → YAML** и перезапустите Home Assistant. В списке
действий должны появиться `rest_command.autopass_check_plate` и
`rest_command.autopass_frigate_event`.
